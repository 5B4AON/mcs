Getting only ~100–200 mV? Most sound cards have AC coupling capacitors that block DC — the output decays to near-zero regardless of amplitude setting. This is a hardware limitation, not a software issue (the digital output is already at 0 dBFS full scale).

→ Switch to AC Tone mode for AC-coupled sound cards. A square wave passes through coupling caps at full amplitude (1–2 V peak). The opto LED half-wave rectifies it automatically; add a Schottky bridge rectifier (4× BAT43) for full-wave rectification and maximum drive current.

Low voltage (<2V peak)? Most headphone outputs are only 1–2V. With an opto LED Vf of ~1.1V, there is very little headroom.
• Remove any series resistor — the sound card’s output impedance (10–50Ω) already limits current. Connect the LED directly.
• Bridge rectifier: Use 4× small-signal Schottky diodes (e.g. BAT43, Vf≈0.3V) in a full-bridge to use both half-cycles, doubling average current. This is the most effective approach for low-voltage outputs.
• Transistor buffer: Use a 2N2222/BC547 with base via 1kΩ from the audio output, collector → opto LED → 5V supply, emitter → GND. This uses external power and works reliably at any audio voltage.

@if (isSectionExpanded('prosigns')) {
Prosigns (procedural signals) are special multi-character morse sequences transmitted without inter-character spacing. In International Morse, some prosigns share identical patterns with punctuation marks, creating ambiguity in text-based displays.

When this setting is enabled, the decoder and encoder fullscreen conversation logs will display prosign names (e.g., <AR>) instead of the underlying punctuation characters (e.g., +). This improves clarity when communicating procedural signals.

Punctuation Prosign Morse Pattern Meaning
+ <AR> .-.-. End of message
& <AS> .-... Wait
= <BT> -...- Break / new paragraph
( <KN> -.--. Go ahead (specific station)

Note: This translation is for display purposes only. Behind the scenes, the application continues to send and receive punctuation characters for efficiency. The conversion happens in real-time during rendering. This applies to the decoder and encoder fullscreen conversation windows only — main screen log panels are not affected.
}
