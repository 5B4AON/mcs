11.1  Common Use Cases

Practice — Learning Morse Code

Use the encoder to send yourself test messages. Type words and listen to the Morse audio through your speakers. Try to decode by ear before looking at the Decoded Output. Gradually increase the WPM as you improve.

On-Air CW — Computer-Keyed Transmission

Connect a USB sound card to your radio via an optocoupler, or use a USB-serial adapter. Type messages in the encoder and let the computer send perfect CW. Use the CW audio input to decode incoming signals.

On-Air CW — Manual Key with Computer Decode

Build the pilot tone circuit and connect your straight key. The computer decodes your sending and displays it. Incoming CW from the radio is fed into the CW audio input and decoded alongside. Both sides of the QSO appear on screen with RX/TX colour coding.

Contest / POTA / SOTA

Pre-type your exchange (e.g. 5NN 042) and send it with one keypress using "Send on Enter" mode. The keyboard keyer handles freeform Morse between exchanges.

SDR + Virtual Cable — All-Software Station

Route audio from your SDR application through a virtual audio cable into Morse Code Studio. Decoded CW appears live. Use the encoder + serial output to key your SDR transceiver. No physical audio cables needed.

Tablet / Phone Operating

Enable the touch keyer in Settings and open the fullscreen decoder. Use the on-screen touch buttons to send CW in straight key or paddle mode. Enable Vibration in the Outputs tab for haptic feedback as you key (Android only). The touch keyer works well on tablets propped up next to a radio.

11.2  Troubleshooting

ProblemSolution
"Start Audio" does nothing Check that you allowed microphone permission. Some browsers require HTTPS — make sure you are not on an insecure HTTP page.
Key level meter stays at zero Verify the pilot tone output channel and device. Check external circuit wiring. Try pilot amplitude 1.0. Run the calibration sequence.
CW level meter stays at zero Check input device and channel selection. Verify the CW frequency matches your radio's sidetone. Try disabling Auto Threshold and setting a low manual threshold.
Decoder shows wrong characters Reset calibration (Reset RX Cal / Reset TX Cal) and set the Decoder WPM close to the actual sending speed. Increase debounce if you see extra dits from contact bounce.
Opto does not key the radio Check wiring polarity. Try DC mode if AC doesn't work (or vice versa). Set amplitude to 1.0. Use "Test Output". Consider a transistor buffer if sound card voltage is too low.
Serial port not available Web Serial API requires Chrome or Edge on a desktop operating system (Windows, macOS, Linux or ChromeOS). Click "Add Serial Port" to grant access the first time — Chrome remembers granted ports across sessions. Android is not supported — Chrome on Android does not provide access to USB-serial adapters even via USB OTG.
Channel conflict warning Two outputs are on the same channel on the same device. Change one to a different channel or device, or disable it.
Audio cuts out or is choppy Close other browser tabs using audio. Try a different browser. On mobile, keep the screen on — some devices throttle background audio.
Mouse keyer not responding Ensure the mouse keyer is enabled in Settings and at least one button is assigned an action other than "None".
Touch buttons not visible Enable the touch keyer in Settings. Touch buttons appear in the fullscreen decoder view. The straight-key sprite button on the main screen is separate — it can be toggled in Settings → Other → Straight-Key Sprite.
Vibration not working Haptic vibration requires Android with Chrome, Firefox or Edge. iOS Safari does not support the Vibration API. Ensure the Vibration toggle is enabled in the Outputs tab of Settings. Some devices may have vibration disabled in system settings.

11.4  Running Multiple Tabs

You can open Morse Code Studio in multiple browser tabs simultaneously. Each tab runs its own independent instance — audio processing, encoder, decoder and keyer state are completely isolated.

Fully Independent (no interference)

  • Audio — Each tab creates its own Web Audio contexts, worklet nodes and oscillators. There is no cross-tab conflict.
  • Encoder / Decoder — All runtime state lives in per-tab Angular services. Nothing leaks between tabs.
  • Firebase RTDB — Each tab opens its own connection. Two tabs listening on the same channel will both receive the data (by design).

Shared Resources

  • Settings & Profiles — Stored in localStorage, which is shared across all tabs on the same origin. Changes saved in one tab are not automatically reflected in another tab until it is reloaded. Saving settings in two tabs at the same time may cause one save to overwrite the other.
  • Web Serial (WinKeyer) — A serial port can only be opened by one tab at a time. If one tab already holds the port, another tab will fail to open it.
  • Service Worker — The Angular service worker is shared across tabs but only caches assets — it does not relay app-level messages, so there is no interference.

11.5  About

Morse Code Studio was conceived on 22 February 2026 (World Scout Day) as a companion to a small hardware circuit designed as a Morse code teaching aid for Jamboree On The Air (JOTA) 2026. Interfacing that circuit with a PC revealed a gap: no existing software offered an easy-to-implement, easy-to-deploy and easy-to-use solution in a single package. Drawing on prior Angular experience and the advantages of Progressive Web Apps, a browser-based prototype was built using HTML5 Web APIs — Web Audio, Web Serial, Web MIDI and others — whose capabilities, discovered through iterative experimentation, gradually shaped the feature set seen today. As the responsive nature of Angular applications was explored further, the interface progressively adopted a mobile-first design philosophy, resulting in an application that is equally comfortable on a phone as it is on a desktop.

What began as a focused tool for displaying decoded CW on a large screen evolved into a versatile, multi-purpose Morse code workbench — a Swiss Army knife of encoding, decoding, keying and interfacing utilities that can serve a wide range of use cases well beyond the original scope. Recognising that others may wish to adapt, enrich and expand this foundation, the project was released to the open-source community under the GNU General Public License v3.0.
