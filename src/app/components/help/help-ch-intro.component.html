1.1  What Is Morse Code Studio?

Morse Code Studio is a browser-based tool for sending and receiving Morse code (CW). It runs entirely in your web browser — there is nothing to install. It works on Chrome, Edge and other Chromium-based browsers on Windows, macOS, Linux, Android and iOS.

Core capabilities:

  • Decode — Listen to incoming Morse from a key or radio and display decoded text on screen in real time.
  • Encode — Convert typed text into perfectly timed Morse code audio and key signals.
  • Key — Send Morse manually using a physical key or paddle connected via an Arduino MIDI interface, USB-serial adapter, computer keyboard, mouse, or touchscreen. Supports straight key and four paddle modes (Iambic A, Iambic B, Ultimatic, Single Lever).
  • Key your radio — Drive your transmitter's keying line through an Arduino MIDI optocoupler, USB-serial adapter (DTR/RTS), WinKeyer, or sound card optocoupler (experimental).

Connecting Your Key and Radio

Several hardware interfaces are supported for physical key input and transmitter keying output, each with different trade-offs:

  1. Arduino MIDI (recommended) — A simple Arduino Pro Micro enumerates as a standard USB MIDI device (no drivers needed) and provides both paddle/key input and optocoupler keying output over a single USB connection. The Web MIDI API delivers input events asynchronously, so timing is crisp even at high WPM. Crucially, MIDI works when the browser is in the background or minimised, making it the most practical interface for on-air use. Ready-made sketches and wiring diagrams are in the project's arduino/ folder.
  2. USB-serial adapter (second choice) — Significantly cheaper (often under $2) and provides both input (DSR/CTS/DCD/RI for key/paddle reading) and output (DTR/RTS for transmitter keying). Like MIDI, serial works in the background. The trade-off is polling-based input (up to ~15 ms latency vs. MIDI's near-zero) and desktop-only support (no Android/iOS).
  3. Keyboard, mouse, and touch — No extra hardware needed, convenient for practice and portable use. The main limitation is that the browser tab must be in focus to receive these events, so they are not suitable for background operation.
  4. Sound card (experimental) — A sound card optocoupler can key the transmitter via the headphone jack, and microphone input with ultrasonic pilot-tone detection can read key closures. The keying input side is experimental and sensitive to audio hardware latency; MIDI or serial is more robust for key input. CW tone decoding of received audio (the primary decode path) works well and is not experimental.

Everything happens in real time with sub-millisecond precision, suitable for on-air CW operating at any practical speed (5–50+ WPM).

1.2  Getting Started

Follow these steps to get up and running:

  1. Click "Start Audio" — The browser needs your permission to use the microphone and speakers. A dialog will appear the first time.
  2. Open Settings — Click the gear icon (⚙) in the header bar to open the Settings modal where you configure inputs, outputs and keyer options.
  3. Choose your inputs — Enable the input method(s) you want: MIDI Input (Arduino paddle/key), Serial Input (USB-serial adapter), keyboard, mouse, or touch keyers. For CW decoding from a radio, enable the CW Tone Detector.
  4. Choose your outputs — Enable the output(s) you need: MIDI Output (Arduino optocoupler), Serial Port (DTR/RTS), WinKeyer, Sidetone (speaker), and/or Optocoupler (sound card, experimental).
  5. Save Settings — Click "Save Settings" at the bottom. Your settings are saved per device configuration and persist across sessions.
Quick Start — Practice decoding

Click Start Audio, open the Encoder panel, type some text and press Enter. Morse audio plays through your speakers while the Decoder displays the decoded text in real time. This is a great way to practise reading Morse code by ear.
