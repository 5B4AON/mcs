9.1  What Is a PWA?

Morse Code Studio is a Progressive Web App (PWA) — a website that can be installed on your device like a native app, complete with a home-screen icon, its own window and the ability to work fully offline. A service worker caches the application code and core assets on first visit; other resources such as help pages and icons are cached on demand. Once cached, subsequent launches are instant with or without an internet connection.

9.2  Installing on Mobile

Android (Chrome / Edge / Brave)

  1. Open the app in Chrome.
  2. Tap the three-dot menu (⋮) → "Install app" (or "Add to Home screen").
  3. Tap "Install" to confirm.

iOS / iPadOS (Safari only)

  1. Open the app in Safari.
  2. Tap the Share button () → "Add to Home Screen" → "Add".

The app icon appears alongside your other apps and launches in a standalone window.

9.3  Installing on Desktop

In Chrome, Edge or another Chromium browser:

  • Click the install icon (⊕) in the address bar, or
  • Chrome: Menu → Save and Share → Install Morse Code Studio…
  • Edge: Menu → Apps → Install Morse Code Studio

The app opens in its own window without browser chrome, giving you more screen space.

9.4  Offline Benefits for Field Operation

Once cached, no internet connection is needed. This is especially valuable for amateur radio field operation:

  • POTA / SOTA / Field Day — Decode, encode and key CW from remote locations with no cellular or Wi-Fi coverage.
  • Emergency comms — One fewer dependency on external infrastructure when networks are down.
  • Instant startup & battery savings — Everything loads from local cache; no network traffic means longer battery life.
  • No account or subscription — Free, open-source and runs entirely on-device with no data sent anywhere.
  • Any device you already carry — Your phone or tablet becomes a capable CW tool without extra equipment.

9.5  Updating the App

When you open the app while online, the service worker checks for updates automatically. New versions download in the background and activate the next time you restart the app — no reinstall needed.

9.6  Screen Wake Lock

Mobile devices and laptops dim and lock the screen after a period of inactivity. During a CW session — especially when decoding or waiting for traffic — you may not be touching the screen for extended periods, yet you still need the display to stay on.

Morse Code Studio can use the browser's Screen Wake Lock API to prevent the device from turning off the display while the app is in the foreground. This is particularly useful for:

  • Decoding sessions — watching incoming CW text without needing to tap the screen periodically.
  • Keyer practice — using a paddle or straight key connected via MIDI or serial while the phone sits on the desk.
  • Field operation — keeping the screen readable in sunlight without it dimming mid-QSO.

Enabling Screen Wake Lock

Open Settings → Other tab and toggle Screen Wake Lock on. The setting is saved per device profile and persists across sessions.

How It Works

  • When enabled, the app requests a wake lock as soon as it starts.
  • If you switch to another app or tab, the lock is automatically released by the browser. When you return, the app re-acquires it.
  • Disabling the toggle releases the lock immediately.
  • The app indicates whether the wake lock is currently active next to the toggle.

Browser Support

The Screen Wake Lock API is supported in Chrome, Edge, Firefox 126+ and Safari 16.4+ (including iOS). If the browser lacks support, a note is shown when the card is expanded.
