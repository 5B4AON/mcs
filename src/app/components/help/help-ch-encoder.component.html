3.1  Sending Modes

The encoder converts typed text into perfectly timed Morse code. Audio plays through the sidetone speaker and key signals are sent through every enabled output (optocoupler, serial port).

ModeHow it worksBest for
Send on Enter Type your full message, then press Enter to send it all at once. Composing careful messages, contest exchanges
Live Each character is sent immediately as you type it. Casual QSOs, real-time ragchew
Copy Practice The app generates a random sequence, plays it as Morse, and you type what you hear. After the round, per-character feedback shows your accuracy. CW copy training, building head-copy speed

Copy Practice mode appears in the encoder mode dropdown when the Copy Practice card is enabled in Settings → Other. See § 3.5 for full details.

Transmit control: Click the transmit button (radio-wave icon) or press Enter in “Send on Enter” mode to begin sending. Click the button again to stop. In the fullscreen encoder view, pressing Escape also aborts transmission.

3.2  Encoder Speed

Adjust the Encoder WPM pill on the main screen to set your sending speed. The encoder uses standard PARIS timing (one PARIS word = 50 dit units).

The encoder WPM is independent of the keyer WPM and both decoder WPMs. This lets you compose text messages at one speed while manually keying at another.

3.3  Fullscreen Encoder

Click the ⛶ button in the main toolbar and choose Fullscreen Encoder View to open a fullscreen view. The conversation history is displayed alongside your encoder buffer so you can read and type in one place. In this view, every keystroke is captured directly — you do not need to click a text box.

The fullscreen encoder has its own independent text buffer, separate from both the main encoder display and the fullscreen decoder. Clearing the conversation in one window does not affect any other. See § 2.5 for details on the clear-menu options.

On touch-capable devices, a floating keyboard button appears in the bottom-right corner. Tap it to open your device’s on-screen virtual keyboard for text entry. Tap again to dismiss.

Example — Calling CQ

Set Encoder WPM to 20. Type CQ CQ CQ DE G3XYZ G3XYZ K and press Enter. The encoder sends perfect Morse at 20 WPM through your sidetone and any enabled key outputs to key your transmitter.

3.4  Prosign Support

You can type prosigns (procedural signals) directly in the encoder buffer using angle-bracket notation. For example, typing HELLO<SK> will send "HELLO" followed by the SK prosign (end of contact). Prosigns are recognized as single tokens and transmitted without inter-character spacing.

Supported prosign patterns:

  • <AR> — End of message
  • <AS> — Wait
  • <BT> — Break; pause
  • <KN> — Invitation to named station only
  • <KA> — Starting signal
  • <SK> — End of contact
  • <SOS> — International distress signal
  • <BK> — Break (invitation to transmit)
  • <HH> — Error; correction follows

Note: Some prosigns (AR, AS, BT, KN) share identical morse patterns with punctuation marks (+, &, =, (). You can type either the prosign pattern or the punctuation character — both will produce the same morse output. Enable Show Prosigns in Settings → Other to display prosign names instead of punctuation in the conversation logs.

When using WinKeyer output, clashing prosigns are automatically converted to their punctuation equivalents since the device only supports ASCII characters. Non-clashing prosigns (<SK>, <SOS>, <BK>, <HH>) are skipped in WinKeyer output.

3.5  Copy Practice

Copy Practice turns Morse Code Studio into a self-study CW trainer. Enable the feature in Settings → Other → Copy Practice, then select Copy Practice from the encoder mode dropdown on the main screen or in the fullscreen encoder toolbar.

How It Works

  1. The app generates a random sequence (characters, words, or callsigns) and feeds it through the encoder so you hear it as Morse audio.
  2. Depending on the feedback mode, the decoded text may be hidden, blurred, or visible while you type what you hear.
  3. After the sequence finishes, the app compares your typed input to the reference using a Longest Common Subsequence (LCS) alignment and highlights each character as correct, incorrect, or missed, with an overall accuracy percentage.

Content Modes

ModeDescription
Random Characters Groups of random characters drawn from a configurable pool of letters (A–Z), digits (0–9), and/or punctuation. Group size is adjustable from 1 to 5 characters.
Random Words Common English words, filterable by length (3, 4, and/or 5 letters). Words are drawn with a frequency bias toward more common words and avoid repeats within a session.
Random Callsigns Realistic amateur radio callsigns generated from an international prefix database with random suffixes — great for contest practice.

The Count setting (1–10) controls how many groups, words, or callsigns are generated per round.

Feedback Modes

ModeDescription
Listen Only Audio plays and decoded characters appear in the output area. After the sequence finishes, type what you heard and submit for scoring.
Blurred Decoded characters appear blurred. Use the reveal button to peek. After the sequence finishes, type and submit for scoring.
Type-Along Decoded characters are blurred while you type in real time as the Morse plays. When the sequence finishes, press the validate button (checkmark) to see per-character feedback.

Pipeline

The Pipeline selector controls which outputs carry the practice audio:

  • Local audio only — plays through the sidetone speaker only. Serial, MIDI, RTDB, and vibration outputs are not driven. Best for solo practice without keying a transmitter.
  • Full pipeline — sends characters through the complete output chain, including serial, MIDI, RTDB, and vibration. Useful when practising with external keying hardware or sharing practice sessions over Firebase RTDB.

Additional Settings

  • Direction — controls whether practice characters appear as RX or TX in the conversation view and colour tagging.
  • Name / Colour — optional display name and colour for practice characters in the fullscreen conversation view.

Fullscreen Practice UI

In the fullscreen encoder view, practice mode replaces the normal transmit button with playback controls:

  • Play (▶) — starts a new round.
  • Pause (▮▮) — pauses playback mid-sequence (finishes the current character).
  • Resume (▶) — continues from where you paused.
  • Next (▶) — when a round is finished, generates and starts a new sequence.
  • Validate (✓) — in type-along mode, compares your typed input to the reference and shows per-character feedback.

After scoring, the reference text is displayed with each character colour-coded — green for correct, red for incorrect, and grey for missed — followed by an accuracy percentage.

Example — Beginner Copy Practice

Enable Copy Practice in Settings → Other. Set Content to Random Characters, Count to 5, Group size to 1, Feedback to Listen Only, and Encoder WPM to 15. Open the fullscreen encoder, select Copy Practice mode from the kebab menu, and press Play. You will hear five individual characters. After playback finishes, type the letters you heard and press Enter to see your accuracy. Increase Count, Group size, or WPM as you improve.

Example — Contest Callsign Practice

Set Content to Random Callsigns, Count to 3, Feedback to Type-Along, and Encoder WPM to 25. Open the fullscreen encoder and press Play. Type each callsign as you hear it. When the sequence finishes, press the validate button to see which characters you got right.
