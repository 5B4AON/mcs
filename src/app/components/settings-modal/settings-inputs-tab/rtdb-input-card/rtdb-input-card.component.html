Subscribe to a Firebase Realtime Database channel to receive morse letters from remote stations. Letters arrive as already-decoded characters and are fed into the encoder as if typed on the keyboard.

Both sender and receiver must share the same Channel Name and Channel Secret. Only the last letter per channel is stored — no chat history is retained.

Requires an internet connection. Offline caching is disabled.
Each character sent via RTDB carries the WPM speed that was used to generate it. By default, received characters are played back at the sender's original speed for realistic timing. Enable this option to ignore the remote WPM and use your local Encoder WPM instead.
