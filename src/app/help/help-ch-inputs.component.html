6.1  Straight Key via Mic (Pilot Tone) Experimental

This input method connects a physical Morse key (straight key or bug) through the microphone input of your sound card using an ultrasonic pilot tone. This feature is experimental — AC coupling behaviour varies across sound cards and USB audio adapters, and results may be unreliable on some hardware.

How It Works

  1. The application sends a constant high-frequency tone (default 18 kHz — inaudible to most people) out of one audio channel to a simple external circuit.
  2. The circuit routes the tone through your Morse key and into the mic input.
  3. When the key is closed, the tone reaches the microphone. When the key is open — silence. A Goertzel filter detects the tone with high precision.

Settings

  • Mic Device — The microphone or sound card input that receives the pilot tone.
  • Pilot Output Device — The sound card that generates the pilot tone. Can be the same card or a different one.
  • Pilot Output Channel — Left or Right. Must not conflict with sidetone or opto on the same device.
  • Pilot Frequency — Default 18,000 Hz (above human hearing). Lower this if your sound card does not pass 18 kHz cleanly.
  • Pilot Amplitude — Volume of the pilot tone. Usually 1.0 is best for reliable detection.
  • Threshold — Detection threshold. Use the Calibrate buttons (described below) to set automatically.
  • Debounce — Minimum time (ms) between state changes. Filters contact bounce from mechanical keys. 5–15 ms is typical.
  • Invert key sense — Swap key-open/key-closed. Useful if your wiring is inverted.
  • Used for — Decoder source (RX or TX). See Chapter 5: Decoder Source Routing.
Calibrating the threshold

1. Click Start Audio. 2. Ensure your key is open (not pressed) and click 1. Key Open — wait for the measurement. 3. Close the key (press and hold) and click 2. Key Closed. 4. Click Apply threshold. The threshold is now set midway between the open and closed levels.

6.2  Straight Key via Audio Channel (CW Tone)

This input decodes CW audio tones directly — no pilot tone circuit needed. Feed audio from a radio receiver, virtual audio cable, or any source carrying CW tones into a sound card input.

How It Works

A narrow-band Goertzel filter is tuned to the configured CW frequency (default 600 Hz). When the tone amplitude exceeds the threshold, the key is considered down.

Settings

  • Audio Input Device — The sound card receiving radio audio.
  • Input Channel — Left or Right.
  • CW Tone Frequency — Match your radio's sidetone / BFO offset. Typically 500–700 Hz.
  • Bandwidth — Width of the detection filter in Hz. Narrower = more selective but needs more precise frequency setting.
  • Auto Threshold — When enabled, the detector automatically tracks the noise floor and signal level, adjusting the threshold dynamically. Recommended for most use.
  • Manual Threshold — Available when Auto Threshold is off. Set by hand using the slider for fixed-level signals.
  • Debounce — Filters brief noise spikes. 5–10 ms is typical for clean signals.
  • Used for — Decoder source (RX or TX). See Chapter 5: Decoder Source Routing.
Example — Decoding with a virtual audio cable

On Windows, install "VB-Cable" or "Virtual Audio Cable". Route your SDR receiver's audio output to the virtual cable. In Morse Code Studio, select the virtual cable as the Audio Input Device. Set the CW frequency to match your SDR's BFO offset. Enable Auto Threshold. The decoder now shows decoded CW text live.

6.3  MIDI Input

Connect a MIDI pedal, foot switch, keyboard, or any MIDI controller to your computer and use it as a morse key or paddles. MIDI input has a key advantage over keyboard and mouse input: it works even when the app is in the background, another window is focused, or the screen is locked / showing a screensaver.

How It Works

The Web MIDI API delivers note-on and note-off messages from MIDI hardware at the browser level, independent of DOM focus. Morse Code Studio maps specific MIDI notes to keyer actions — straight key, dit paddle, or dah paddle — using the same keyer engine as the keyboard and mouse keyers.

Settings

  • MIDI Device — Select a specific MIDI input, or leave on “Any” to accept messages from the first available device.
  • MIDI Channel — Filter by MIDI channel (1–16), or “Omni” to accept all channels.
  • Straight Key — Click the capture button and press a key/pad on your MIDI controller. That note will act as a straight key (press = key down, release = key up).
  • Dit Paddle — Capture a MIDI note for the dit paddle. Used with iambic keyer modes.
  • Dah Paddle — Capture a MIDI note for the dah paddle.
  • Reverse Paddles — Swap dit and dah paddle assignments.
  • Paddle Mode — Iambic A, Iambic B, Ultimatic, or Single Lever (shared with keyboard and mouse keyers).
  • Used for — Decoder source (RX or TX). See Chapter 5: Decoder Source Routing.
Assigning MIDI notes

1. Enable MIDI Input and expand the settings card.
2. Click the “Straight Key” capture button — it shows “Press a MIDI key…”.
3. Press the desired key or pedal on your MIDI controller.
4. The button updates to show the captured note name (e.g. “C4 (60)”).
5. Repeat for Dit and Dah paddles if using paddle mode.
6. Click the × button next to any assignment to clear it.

Background operation

Unlike keyboard or mouse input, MIDI messages are delivered by the browser even when the tab is not focused. Combined with the audio keep-alive that Morse Code Studio maintains, your MIDI key/pedal continues to work during screensaver, lock screen, or while another application has focus.

MIDI input also bypasses the keyboard keyer’s enabled/disabled state. When a text field has focus, the keyboard keyer is automatically disabled to avoid stray characters, but MIDI keying continues uninterrupted. This means you can type in the encoder while still keying via your MIDI pedal or controller.

Full-duplex echo prevention & break-in (QSK)

When MIDI input and output share the same bus or device, the app automatically prevents feedback loops:

  • Echo suppression — while a MIDI output note is physically held, all incoming note-on messages are treated as loopback echoes and dropped.
  • Break-in — when a valid (non-echo) MIDI input note arrives, the entire MIDI output queue is flushed, held notes are released, and output is muted. New characters from the encoder are dropped until the remote party goes silent for a full word gap, at which point output resumes automatically.
  • Reverse-path tagging — characters decoded from MIDI input carry a fromMidi tag and are never forwarded back to MIDI output.
Browser compatibility

Web MIDI API is supported in Chrome, Edge, and Opera. Firefox and Safari do not support it. MIDI permission is requested transparently when you click Start Audio — no separate permission step is needed.

Arduino Pro Micro as a MIDI key/paddle

The project includes ready-made Arduino sketches (in the arduino/ folder of the source repository) that turn a Pro Micro into a USB MIDI device with straight key and paddle inputs on pins 2, 3 and 4. Two board variants are provided: ATmega32U4 (classic Pro Micro, using the MIDIUSB library) and nRF52840 (Supermini / nice!nano, using Adafruit TinyUSB). Flash the sketch, connect your key or paddles, and the Arduino appears as a standard MIDI device — no drivers needed. Wiring diagrams and setup instructions are included.
