6.1  Straight Key via Mic (Pilot Tone) Experimental

This input method connects a physical Morse key (straight key or bug) through the microphone input of your sound card using an ultrasonic pilot tone. This feature is experimental — AC coupling behaviour varies across sound cards and USB audio adapters, and results may be unreliable on some hardware.

How It Works

  1. The application sends a constant high-frequency tone (default 18 kHz — inaudible to most people) out of one audio channel to a simple external circuit.
  2. The circuit routes the tone through your Morse key and into the mic input.
  3. When the key is closed, the tone reaches the microphone. When the key is open — silence. A Goertzel filter detects the tone with high precision.

Settings

  • Mic Device — The microphone or sound card input that receives the pilot tone.
  • Pilot Output Device — The sound card that generates the pilot tone. Can be the same card or a different one.
  • Pilot Output Channel — Left or Right. Must not conflict with sidetone or opto on the same device.
  • Pilot Frequency — Default 18,000 Hz (above human hearing). Lower this if your sound card does not pass 18 kHz cleanly.
  • Pilot Amplitude — Volume of the pilot tone. Usually 1.0 is best for reliable detection.
  • Threshold — Detection threshold. Use the Calibrate buttons (described below) to set automatically.
  • Debounce — Minimum time (ms) between state changes. Filters contact bounce from mechanical keys. 5–15 ms is typical.
  • Invert key sense — Swap key-open/key-closed. Useful if your wiring is inverted.
  • Used for — Decoder source (RX or TX). See Chapter 5: Decoder Source Routing.
Calibrating the threshold

1. Click Start Audio. 2. Ensure your key is open (not pressed) and click 1. Key Open — wait for the measurement. 3. Close the key (press and hold) and click 2. Key Closed. 4. Click Apply threshold. The threshold is now set midway between the open and closed levels.

6.2  Straight Key via Audio Channel (CW Tone)

This input decodes CW audio tones directly — no pilot tone circuit needed. Feed audio from a radio receiver, virtual audio cable, or any source carrying CW tones into a sound card input.

How It Works

A narrow-band Goertzel filter is tuned to the configured CW frequency (default 600 Hz). When the tone amplitude exceeds the threshold, the key is considered down.

Settings

  • Audio Input Device — The sound card receiving radio audio.
  • Input Channel — Left or Right.
  • CW Tone Frequency — Match your radio's sidetone / BFO offset. Typically 500–700 Hz.
  • Bandwidth — Width of the detection filter in Hz. Narrower = more selective but needs more precise frequency setting.
  • Auto Threshold — When enabled, the detector automatically tracks the noise floor and signal level, adjusting the threshold dynamically. Recommended for most use.
  • Manual Threshold — Available when Auto Threshold is off. Set by hand using the slider for fixed-level signals.
  • Debounce — Filters brief noise spikes. 5–10 ms is typical for clean signals.
  • Used for — Decoder source (RX or TX). See Chapter 5: Decoder Source Routing.
Example — Decoding with a virtual audio cable

On Windows, install "VB-Cable" or "Virtual Audio Cable". Route your SDR receiver's audio output to the virtual cable. In Morse Code Studio, select the virtual cable as the Audio Input Device. Set the CW frequency to match your SDR's BFO offset. Enable Auto Threshold. The decoder now shows decoded CW text live.
