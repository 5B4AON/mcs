8.1  Sound Card Channel Routing

Each stereo sound card has two channels: Left and Right. Morse Code Studio routes each output (sidetone, opto, pilot) to a specific channel on a specific device. This lets you use a single sound card for multiple purposes.

The rule: Two outputs must not share the same channel on the same device. If there is a conflict, the application shows a warning.

Single sound card — choose 2 of 3: A stereo card has only two channels but there are three possible outputs (sidetone, opto, pilot). You can enable any two. To use all three, add a second USB sound card.

8.2  Device Profiles

When you save settings, they are stored against the current set of audio devices connected to your computer. This is called a device profile.

If you later connect a different set of devices (e.g. plugging in a USB sound card at home vs. a portable station), the application detects the change and loads the matching profile automatically. Each hardware configuration is remembered separately.

If no profile matches the current devices, the application creates a new profile with default settings and shows a yellow banner asking you to review and save.

Example — Home vs. portable

At home: desktop PC with a built-in sound card plus a USB card for keying. Portable: laptop with only its built-in audio. Each setup has different device IDs, so Morse Code Studio saves and loads the correct settings automatically.

8.3  Fullscreen Mode

Both the Decoder and Encoder panels have a ⛶ button. Clicking it opens a fullscreen overlay optimised for operating.

Independent Display Buffers

The application maintains four independent FIFO text buffers:

  • Main Decoder — the Morse Decoder panel on the main screen.
  • Main Encoder — the sent-character display in the Encoder panel.
  • Fullscreen Decoder — the conversation log in the fullscreen decoder view.
  • Fullscreen Encoder — the conversation log in the fullscreen encoder view.

Each buffer receives text from the same data sources (decoded characters, sent encoder characters, and Firebase RTDB input) but can be cleared independently. Pressing the Clear button (🗑) opens a context menu with This log (clears the current buffer only) and All logs (clears all four buffers at once). The encoder’s clear menu adds a Text area option to empty the text-entry field without affecting any log.

Buffers are capped at approximately 5 000 characters (~2× a full screen of text). When the limit is reached, the oldest characters are silently discarded (FIFO). Both fullscreen buffers persist across close/reopen — you will not lose your conversation history by switching between views.

Decoder Fullscreen

Large, scrollable decoded text with RX/TX colour coding. The toolbar provides WPM adjustment (RX, TX, Keyer), calibration reset buttons, and level meters for the Key and CW inputs. When Firebase RTDB input or output is active, a small RTDB status indicator (wifi icon) turns green in the toolbar.

Encoder Fullscreen

Conversation history plus your encoder buffer inline. Every keystroke is captured directly — no need to click a text box. The toolbar shows encoder WPM, sending mode, and TX ON/OFF controls. On mobile/tablet devices, a floating keyboard button (⌨) appears to toggle the on-screen virtual keyboard. The RTDB status indicator is also visible here.

Responsive Toolbars

On narrow screens, the main controls bar and each fullscreen modal toolbar automatically collapse overflow controls into a kebab menu (⋮). Essential buttons — Start Audio on the main screen, and Close / Clear in fullscreen modals — remain visible at all times. Tap the ⋮ button to access the remaining controls in a dropdown. A Help (?) entry at the bottom of each kebab menu opens the built-in help system. The toolbar restores to its full inline layout when the window is wide enough.

Display Customisation

The following display settings are available in both fullscreen views and are persisted independently of device profiles:

  • Font size — Adjustable from 24 to 184 px.
  • Bold — Toggle bold text.
  • Line spacing — Vertical spacing between conversation lines.
  • Colours — Custom colours for RX text, TX text, buffered text, currently-sending character, and background.
